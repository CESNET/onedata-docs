This guide covers everything you need before you can start storing, sharing, and collaborating in Onedata.

## Prerequisites

Before you begin, make sure you have:

- An **e-INFRA CZ account** that is part of the e-INFRA organization.
  - Not a member yet? [Request access here](https://docs.account.e-infra.cz/en/docs/access/account).
- At least one **Space** assigned to you, or a request in progress to create one.

> **Note:** A Space is your personal or project workspace in Onedata. It is not created automatically.

---

## Procedure

### 1. Choose how you want to access Onedata

Onedata offers several ways to work with your data. Pick the one that fits your workflow.

| Method | Best for | Requires |
|---|---|---|
| **Web Portal** | Browsing, sharing, managing files via browser | Any modern web browser |
| **Oneclient** | Command-line workflows, HPC jobs, scripting | Linux system + package installation |

#### Web Portal (recommended for beginners)

The easiest way to get started. No installation needed — just log in through your browser.

- **URL:** [https://onedata.e-infra.cz](https://onedata.e-infra.cz)
- **Best for:** browsing files, managing shares, inviting collaborators, basic uploads and downloads

> **Tip:** Allow pop-ups in your browser for proper file handling in the Web Portal.

#### Oneclient (for advanced users)

Mount your Onedata Space as a local directory on a Linux computer or HPC cluster. Once mounted, you work with files as if they were on your local disk.

- **Best for:** command-line workflows, HPC jobs, scripting, and working with large datasets
- **Requires:** Linux with the `oneclient` package installed

See the [Oneclient guide](/docs/how-to/how-to-use-client) for installation and setup instructions.

### 2. Request a Space

Before you can upload or share anything, you need a Space.

- If a colleague already invited you to a Space, you are ready to go.
- If you need a new Space, [request one via email](/docs/how-to/how-to-request-space).

> **Warning:** Spaces are not created automatically. Make sure you have access to at least one Space before trying to upload data.

---

## Sandbox

Sandbox is a special kind of Space, pre-created by default and available to all users. It is intended for testing and experimentation, as the name suggests.

> **Warning:** Please be aware that **everyone** has read and write access to **everything** in Sandbox. Data stored there are therefore ephemeral and public.
  **Do not place anything important or confidential in Sandbox!**

See also [the Sandbox section of the official Onedata documentation](https://onedata.org/docs/25/user-guide/quickstart/#sandbox).
